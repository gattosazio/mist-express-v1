# MISSU Express Backend

**MISSU** stands for **Institutional Model for Integrated Safety & Security Understanding.**

MISSU is a voice-enabled, policy-aware institutional assistant designed for enterprise environments where policy mistakes can create legal, safety, operational, or security risks. Instead of behaving like a generic chatbot, MISSU retrieves policy evidence, generates grounded answers, speaks them in real-time, and securely records interactions for strict auditability.

---

## 🧠 System Architecture & Workflow

MISSU acts as a secure backend participant in a real-time audio room. The current voice and data flow operates as follows:

1. **User Voice** streams into LiveKit.
2. **LiveKit** routes audio to the MISSU backend worker.
3. **Deepgram STT** transcribes the speech to text.
4. **Transcript Filtering** cleans and validates the input.
5. **Semantic RAG Retrieval** queries the pgvector database via Ollama embeddings.
6. **Grounded LLM Answer** is generated by Groq based strictly on retrieved policy.
7. **Deepgram TTS (Aura)** converts the text response back to raw audio.
8. **LiveKit Playback** streams the audio back to the user.
9. **PostgreSQL Audit Log** permanently records the entire interaction.

✨ Core Features
Real-Time Voice Assistant: Joins LiveKit rooms, listens, transcribes, grounds answers, and speaks.

Advanced Transcript Filtering: Includes wake-word support, minimum length checks, duplicate suppression, and cooldown logic.

TTS Playback: Generates Deepgram Aura WAV output, parses to PCM, and streams via LiveKit AudioSource.

Semantic RAG Retrieval: Ingests policies, creates chunks, and utilizes Ollama embeddings/pgvector for nearest-neighbor search (with lexical fallback).

Enterprise Audit Logging: Records participant identity, queries, grounded answers, confidence scores, escalation flags, citations, and metadata.
